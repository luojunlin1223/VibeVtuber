# ADR-003: RVC (Retrieval-based Voice Conversion) for AI Voice Changer

**Date:** 2026-02-18
**Status:** Accepted

---

## Context

The user wanted to implement a voice changer for the VTuber project with the ability to load voice packs from anime characters or internet celebrities. The requirement was to convert the user's live voice into a target character's voice in real time.

Two main categories of solution were considered:

1. **Simple pitch/formant shifting** — libraries like `pyrubberband`, `pyworld`, `praat-parselmouth`
2. **AI voice conversion** — frameworks like RVC, So-VITS-SVC, w2v2-VITS

## Decision

Use **RVC (Retrieval-based Voice Conversion)** as the core voice conversion framework.

Architecture:
- `PythonVoiceChanger/` — standalone Python module separate from `PythonFaceTracker/`
- Real-time audio pipeline: microphone → RVC inference → BlackHole virtual device → OBS/Discord/Unity
- Control panel integration: model files (`.pth`) dropped into `models/`, auto-discovered via `GET /api/voice-changer/models`
- Fallback: `SimpleRVC` (librosa pitch shift) when no model is provided

## Alternatives Considered

| Option | Pros | Cons |
|--------|------|------|
| Simple pitch shift (pyrubberband) | Instant, low latency (<30ms), no model needed | Not true voice cloning; won't sound like target character |
| So-VITS-SVC | Similar quality to RVC | Less community support, fewer pretrained models available |
| RVC | Large community, many pretrained models, good quality | Higher latency (100-200ms), requires HuBERT pretrained model, GPU recommended |
| Phased approach (pitch first, RVC later) | Faster to get running | User explicitly chose to skip the simple version |

## Rationale

- User's core requirement was loading voice packs (anime/VTuber audio sources), not simple pitch shifting
- RVC has the largest ecosystem of community-trained models for anime characters and VTubers
- RVC integrates naturally as a Python subprocess — fits the existing control panel architecture
- Virtual audio output (BlackHole) decouples the voice changer from Unity/OBS, keeping components independent

## Consequences

- **HuBERT pretrained model** (`hubert_base.pt`) must be downloaded separately (~360MB)
- **GPU/MPS acceleration** strongly recommended for real-time latency (<200ms)
- RVC `.pth` model files must be sourced externally (Hugging Face, AI-Hub communities)
- Full RVC inference implementation (HuBERT feature extraction + Faiss retrieval) is currently stubbed — `SimpleRVC` is the active backend until full implementation is completed
- BlackHole virtual audio device required on macOS for routing to OBS/Discord

## Implementation Location

```
PythonVoiceChanger/
├── voice_changer.py        # main entry, CLI, VoiceChanger controller
├── rvc_inference.py        # RVCInference + SimpleRVC + create_rvc_engine()
├── audio_engine.py         # sounddevice real-time audio I/O
├── models/                 # drop .pth files here
└── pretrained/             # hubert_base.pt goes here
```
